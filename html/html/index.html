HODLRdD - Hierarchically off-diagonal low-rank matrix in dimensions. This page hosts the documentation of HODLDdDlib powered by doxygen.

Highlights of HODLDdDlib

  • Black-box and kernel independent
  • Low-rank approximation is through Adaptive Cross Approximation
  • Solution to well-conditioned linear systems

Hyper-parameters for HODLRdD

  1. NDIM = The dimension space of the particles.
  2. Nmax = The maximum number of particles at the leaf level (This decides the depth of the hierarchical tree)
  3. eps_ACA = Accuracy in compression by ACA
  4. SYS_SIZE = Above this size the matrix operations are performed in an memory efficient way
  5. INTERACTION_TYPE_ALLOWED = This represents $d'$ (to decide on which weak admissibility criteria)
The following options are allowed in deciding the hierarchical tree and/or the admissibility criteria
  • In Two dimenstions ($2$ D):
    1. $d' = -1$ $\implies$ $\mathcal{H}$-matrix with standard/strong admissibility criteria $\eta = \sqrt{2}$.
    2. $d' = 0$ $\implies$ Interaction list includes Vertex sharing neighbours, HODLR2D matrix structure.
    3. $d' = 1$ $\implies$ Interaction list includes all non-self clusters. HODLR with quad tree
  • In Three dimensions ($3$ D):
    1. $d' = -1$ $\implies$ $\mathcal{H}$-matrix with standard/strong admissibility criteria $\eta = \sqrt{3}$.
    2. $d' = 0$ $\implies$ Interaction list includes Vertex sharing neighbours, HODLR3D matrix structure.
    3. $d' = 1$ $\implies$ Interaction list includes Vertex sharing and edge sharing neighbours
    4. $d' = 2$ $\implies$ Interaction list includes all non-self clusters. HODLR with oct tree
  • In Higher dimensions:
    1. $d' = -1$ $\implies$ $\mathcal{H}$-matrix with standard/strong admissibility criteria $\eta = \sqrt{NDIM}$.
    2. $d' = 0$ $\implies$ Interaction list includes Vertex sharing neighbours, HODLR $d$ D matrix structure.
    3. $\vdots$
    4. $d' = NDIM-1$ $\implies$ Interaction list includes all non-self clusters. HODLR with $2^{NDIM}$ tree.

Steps to Compile

  • Step 1: usrname_cmd$ git clone https://github.com/SAFRAN-LAB/HODLRdD.git
  • Step 2: usrname_cmd$ cd HODLRdD && cd examples
  • Step 3: Update the `Eigen` path in `Makefile`
  • Step 4: Run the `Makefile` to get the executable
  • Step 5: If you want to get the results for HODLR4D matrix set `atoi(argv[2]) = 0` Then : usrname_cmd$ make clean && make && ./exe 25 0
